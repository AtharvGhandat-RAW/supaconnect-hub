CHAPTER 3
SCOPE OF PROJECT

3.1 INTRODUCTION

The scope of the Attendro project defines the boundaries of the system, outlining what the solution will deliver in terms of functionalities, constraints, and operational capabilities. This project aims to design and develop a comprehensive Portable Wireless Biometric Attendance System that bridges the gap between hardware-based biometric acquisition and cloud-based data management.

The core objective is to eliminate the inefficiencies of manual paper-based attendance and the immobility of traditional wall-mounted biometric systems. The scope encompasses the development of a handheld IoT device based on the ESP32 microcontroller, a robust optical fingerprint sensor (R307), and a modern React.js web dashboard for administration. By leveraging Supabase for real-time database synchronisation, the system ensures that attendance data is instantly available to faculty and administrators, thereby streamlining academic record-keeping processes.

3.2 FUNCTIONAL SCOPE

The functional scope details the specific features and interactions that the system functions will support. The Attendro system is divided into three primary functional modules: the Hardware Device, the Web Interface, and the Database layer.

3.2.1 The Portable Biometric Device

The hardware unit is the primary interface for students. Its functional scope includes:

  • Biometric Acquisition: The device must capture fingerprint minutiae using the R307 optical sensor. It is responsible for verifying the captured print against stored templates on-board or via cloud verification.
  • Wireless Connectivity: Utilising the ESP32’s Wi-Fi capabilities, the device must connect to the institution's network to transmit attendance logs to the server in real-time.
  • User Feedback: A 0.96-inch OLED display provides visual feedback (e.g., "Attendance Marked", "Try Again", "Connecting..."), ensuring the user is aware of the system state.
  • Portability & Power Management: The system runs on a 3.7V Li-ion battery, allowing it to be passed around the classroom. It includes power-management routines to sleep when inactive.

3.2.2 The Web Interface (Dashboard)

The web application serves as the control centre for faculty and administrators. Its key functions include:

  • Faculty Authentication: Secure login functionality for professors to access their specific courses and student lists.
  • Session Control: Faculty can remotely "Start" or "Stop" an attendance session for a specific class. The hardware device will only accept inputs when a session is active.
  • Student Registration: A module to enrol new students, associating their metadata (Name, Roll No.) with their biometric ID.
  • Reporting & Analytics: Generation of attendance reports, identifying absentees, and visualising attendance trends over time.

3.2.3 The Centralized Database

The backend scope involves the management of persistent data:

  • Real-time Synchronisation: The database captures events sent by the ESP32 device immediately.
  • Data Integrity: Ensuring that student records and attendance logs are linked correctly via foreign keys.
  • Storage: Secure storage of user credentials, student profiles, and timestamped attendance logs.

3.3 NON-FUNCTIONAL SCOPE

The non-functional scope defines the quality attributes and constraints under which the system operates.

3.3.1 Reliability

The system is designed to handle temporary network failures. In the event of Wi-Fi disconnection, the device should hold the state or notify the user, ensuring no false positives are recorded. The fingerprint sensor is selected for its high durability and resistance to scratches.

3.3.2 Security

Security is paramount in handling biometric data and student records. The system employs Token-based Authentication (via Supabase Auth) for the web dashboard. Communication between the ESP32 and the cloud is secured over HTTPS/TLS to prevent data interception. Furthermore, fingerprint data is processed as templates rather than raw images, adding a layer of privacy.

3.3.3 Performance

Performance metrics are critical for user acceptance:

  • Matching Speed: The fingerprint identification process is optimised to complete within 1 second.
  • Boot Time: The device boots and connects to Wi-Fi within 5–10 seconds.
  • Dashboard Latency: Updates from the device appear on the faculty dashboard with minimal latency (typically under 2 seconds), facilitated by WebSocket subscriptions.

3.5 FEASIBILITY STUDY

3.5.1 Technical Feasibility

The project is technically feasible as it utilises the ESP32, a mature and widely supported platform in the IoT domain. The R307 sensor communicates via standard UART protocols, and libraries for interface are readily available.

3.5.2 Operational Feasibility

Operationally, the system is intuitive. Faculty members require no coding knowledge. For students, the interaction is limited to placing a finger on the sensor. The device's portability ensures it fits into existing classroom workflows.

3.5.3 Economic Feasibility

Attendro utilises open-source software libraries and affordable off-the-shelf hardware components (approx. total cost < ₹1500), making it a significantly more cost-effective solution than commercial alternatives for educational institutions.
