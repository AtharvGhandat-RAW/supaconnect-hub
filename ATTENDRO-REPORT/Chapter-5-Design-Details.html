5.1 DATABASE ARCHITECTURE

The backend infrastructure is built on Supabase (PostgreSQL). The data model is designed to be relational, ensuring referential integrity between students, sessions, and attendance records. An Entity-Relationship (ER) diagram visualizes these connections.

The `logs` table is the central fact table where records accumulate. It references the `profiles` table to fetch student details dynamically. Row Level Security (RLS) policies are implemented to ensure that only authorized faculty can read sensitive logs.

5.2 HARDWARE INTERFACE DESIGN

The physical design is compact and handheld. The circuit schematic illustrates the interconnections between the Microcontroller (ESP32) and peripheries.

  • Voltage Regulation: The system accepts 5V input via USB-C or Li-ion. The AMS1117 regulator on the ESP32 dev board drops this to 3.3V for the logic level.
  • Communication Bus: To avoid signal collision, the Fingerprint Sensor uses dedicated Hardware Serial (UART2), while the OLED display uses the I2C bus.

5.3 USER INTERFACE (DASHBOARD) DESIGN

The web interface is built using React.js and Tailwind CSS. It is designed to be responsive, working seamlessly on mobile and desktop browsers.

  • Live Feed: A WebSocket subscription to the database allows the dashboard to update in real-time as soon as a student marks attendance.
  • Export Panel: Administrators can filter records by date and download them as CSV or PDF reports for official documentation.
  • Student Profile: Clicking on a student record reveals a detailed history of their attendance percentage.
