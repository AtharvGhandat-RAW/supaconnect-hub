5.1 DATABASE DESIGN

The system utilizes a relational database (PostgreSQL provided by Supabase) to manage entities. The schema is normalized to Third Normal Form (3NF) to reduce redundancy.

5.3 SOFTWARE ARCHITECTURE

5.3.1 Edge Functions

To reduce latency and secure the database, we employ Supabase Edge Functions (Deno/TypeScript):

  • start-session: Validates faculty credentials and creates a new lecture_session entry.
  • mark-attendance: Receives Roll No & Device ID, validates if the student belongs to the current batch, and inserts into attendance_records.

5.3.2 Frontend Structure (React)

The faculty dashboard is component-based:

  • AuthProvider: Manages login state.
  • SessionManager: Handles the "Start/Stop" session logic.
  • LiveAttendance: Subscribes to Supabase Realtime changes to show students popping up as they scan.
