4.1 INTRODUCTION

The efficacy of the Attendro system lies in its systematic workflow that ensures data integrity from the moment a student places their finger on the sensor. The methodology is divided into two distinct operational modes: the Enrollment Phase (one-time setup per student) and the Attendance Phase (recurring utilization).

4.2 ENROLLMENT WORKFLOW

The enrollment process is the foundation of the biometric system. It involves converting a physical fingerprint into a digital template. This phase is typically supervised by a faculty or admin.

  1. Initialization: The device is set to 'Enroll Mode' via the web dashboard.
  2. Scanning: The R307 sensor captures the fingerprint image twice to ensure consistency and high quality. The internal algorithm checks for minutiae points.
  3. Template Generation: The image is not stored. Instead, it is processed into a proprietary hexadecimal character string (Template).
  4. Storage: This template is uploaded to the Supabase Cloud Database along with the student's Roll Number. It is then synced to local device memory for fast offline matching.

4.3 ATTENDANCE WORKFLOW

The core functionality mechanism involves verifying identity within a restricted time window. The algorithm running on the ESP32 ensures that attendance is not just about matching a finger, but about matching it at the right time.

  1. Session Check: The ESP32 periodically (every 5 seconds) queries the cloud for an 'Active Session Token'. If no session is active, the device remains locked.
  2. Biometric Input: A student places their finger. The device searches its local library (1:N matching).
  3. Validation: If a match is found, the system retrieves the associated Roll Number.
  4. Logging: The Roll Number, current Timestamp, and Session ID are bundled into a JSON packet and sent to the `attendance_logs` table in Supabase via secure HTTPS POST request.
  5. Visual Feedback: The OLED displays the student's name and status ("Present").

4.4 HARDWARE ASSEMBLY

The hardware integration follows a modular approach.

  • Controller: The ESP32 acts as the master.
  • Sensor Interface: The R307 fingerprint sensor communicates via UART (Serial2) on pins 16 (RX) and 17 (TX).
  • Display Interface: The OLED display uses the I2C protocol, connected to pins 21 (SDA) and 22 (SCL).
  • Power: A TP4056 charging module manages the Li-ion battery, providing a regulated 5V boost to the sensor and 3.3V to the ESP32.
